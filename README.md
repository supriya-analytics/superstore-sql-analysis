This project contains SQL queries written on a Superstore Sales dataset

## ðŸ—‚ Dataset Columns
Order_Id, Order_date, Ship_mode, Region, Category,  
Price_per_Unit, Quantity, Sales, Profit

## ðŸ“Š SQL Queries & Meaning

1. View all records
SELECT * FROM superstore_sales;
- Meaning:
Displays the complete dataset. Used to understand table structure and data.

2. Filter Orders by Region
SELECT *
FROM superstore_sales
WHERE Region = 'East';
- Meaning:
Fetches only orders from the East region for region-specific analysis.

3. Filter High Sales Orders
SELECT *
FROM superstore_sales
WHERE Sales > 50000;
- Meaning:
Identifies high-value orders contributing significantly to revenue.

4. Top 10 Highest Sales Orders
SELECT *
FROM superstore_sales
ORDER BY Sales DESC
LIMIT 10;
- Meaning:
Shows the top 10 orders based on sales amount.

5. Total Sales of the Store
SELECT SUM(Sales) AS Total_Sales
FROM superstore_sales;
- Meaning:
Calculates total revenue generated by the store.

6. Average Profit per Order
SELECT AVG(Profit) AS Avg_Profit
FROM superstore_sales;
- Meaning:
Finds the average profit earned on each order.

7. Total Sales by Region
SELECT Region, SUM(Sales) AS Total_Sales
FROM superstore_sales
GROUP BY Region;
- Meaning:
Compares sales performance across different regions.

8. Total Profit by Category
SELECT Category, SUM(Profit) AS Total_Profit
FROM superstore_sales
GROUP BY Category;
- Meaning:
Identifies which product category generates the most profit.

9. Regions with High Sales (HAVING)
SELECT Region, SUM(Sales) AS Total_Sales
FROM superstore_sales
GROUP BY Region
HAVING SUM(Sales) > 500000;
- Meaning:
Filters regions where total sales exceed 500,000.
Uses HAVING because the condition is applied on aggregated data.

10. Monthly Sales Report (Date Range)
SELECT SUM(Sales) AS Monthly_Sales
FROM superstore_sales
WHERE Order_date BETWEEN '2024-02-01' AND '2024-02-29';
- Meaning:
Calculates total sales for a specific month.
Useful for monthly performance reporting.
